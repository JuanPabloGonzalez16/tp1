# tp1
Es conveniente incluir el archivo .gitignore en un proyecto porque ayuda a mantener el repositorio limpio y organizado. También evita que se confirmen accidentalmente archivos innecesarios o confidenciales. 
Beneficios de usar .gitignore Evita que se rastreen archivos que no se necesitan, Reduce el desorden, Mejora la colaboración, Optimiza el repositorio, Permite mantener un control más eficiente sobre el proyecto. 
El archivo .gitignore se debe incluir en el directorio raíz del proyecto, o en cualquier otra carpeta del proyecto. Esto le indica a Git qué archivos y carpetas debe ignorar. 
Para configurar el archivo .gitignore, puedes: 
Abrir la terminal
Navegar al directorio del repositorio de Git
Crear un archivo de texto vacío llamado ".gitignore"
Editar el archivo y agregar en cada nueva línea los archivos o carpetas que quieras que Git ignore
